Retail ETL Pipeline project

In this Data Pipeline project, CloudWatch is employed to schedule Lambda functions responsible for scanning an S3 bucket. Upon completion of the scanning process and determination that the required data is fully available, a signal is sent to Apache Airflow. Airflow is then triggered to coordinate the execution of tasks, including parsing requests, running Apache Spark on AWS EMR, and monitoring the process through a watch step. The processed and cleaned data is subsequently stored in the Parquet format on Amazon S3. AWS Glue and Athena are utilized for direct data crawling, facilitating BI downstream usage through efficient querying capabilities.

Weather Streaming Pipeline Project

The project is to develop a comprehensive data processing system that captures and processes real-time weather data for analysis. The system will harness the OpenWeather API to collect data, which will then be routed through Apache NiFi for preprocessing. The processed data will be stored in a MySQL database, with Debezium tracking changes for real-time capture. Apache Kafka will be used to handle the data stream, allowing Apache Spark Streaming to process the data for updates and analytical computations. Apache Hudi will manage the efficient storage and retrieval of this data on AWS S3, ensuring data consistency and accessibility with cost-efftective benefit. Finally, Amazon Athena will be utilized to query the data, facilitating complex analytical tasks, with the results visualized for data-driven decision-making and reporting.

Datawarehouse Optimization

This project seamlessly automates data ingestion from the Google Ads API at specified intervals using Airbyte. The collected data is stored in a PostgreSQL datawarehouse. Subsequently, dbt is employed to perform data transformations tailored to business layer requirements through defined models. Finally, Metabase is utilized to create compelling visualizations based on the refined data, facilitating insightful analysis and decision-making.
